52.1. Examples#

52.1.1. Robber’s cipher#

Let’s start by creating a class for the Robber’s cipher whose instances will hold the vowel that we want to use in a Caesar cipher. Eventually, we’ll also incorporate instance methods that operate on this encapsulated vowel but let’s take it one step at a time, eh.

Let’s call the class RobbersCipherVowel and let’s call the field Vowel. We’ll make the field public so that anyone who has access to an object of type RobbersCipherVowel.

class RobbersCipherVowel
{
  public char Vowel = 'o';
}
The below script needs to be able to find the current output cell; this is an easy method to get it.

We can now instantiate ciphers.

RobbersCipherVowel v1 = new RobbersCipherVowel();
RobbersCipherVowel v2 = new RobbersCipherVowel();

And assign them different vowels.

v1.Vowel = 'a';
v2.Vowel = 'B';

Danger

Wait a minute! B is not a vowel. We’ll have a look at this problem in the chapter on constructors.

Of course we could then rewrite the encode methods that we wrote in the chapter on overloading so that they take a parameter of type RobbersCipherVowel instead of simply a char.

static class RobbersCipher
{
  public static string Encode (char input, RobbersCipherVowel vowel)
    => input switch {
      'B' or 'b' or 'C' or 'c' or 'D' or 'd' or 'F' or 'f' or 'G' or 'g' or 'H' or 'h' or 'J' or 'j' or 'K' or 'k' or 'L' or 'l' or 'M' or 'm' or 'N' or 'n' or 'P' or 'p' or 'Q' or 'q' or 'R' or 'r' or 'S' or 's' or 'T' or 't' or 'V' or 'v' or 'W' or 'w' or 'X' or 'x' or 'Y' or 'y' or 'Z' or 'z'
      => $"{input}{vowel.Vowel}{input}",
    _ => $"{input}"
  };

  publc static string Encode (string input, RobbersCipherVowel vowel)
  {
    string output = "";
    foreach (char letter in input)
      output += Encode (letter, vowel);
    return output;
  }
}

(10,9): error CS1585: Member modifier 'static' must precede the member type and name

The static encode methods still work but now we have to pass them instances of RobbersCipherVowel in order to run them.

Console.WriteLine( RobbersCipher.Encode( "Hello", v1) );

Console.WriteLine( RobbersCipher.Encode( "Hello", v2) );

Seems to work. However, this is not the object oriented way.

Important

In object oriented programming we mix methods and data. We don’t simply store data in object and then pass these objects to methods so that they can extract the data.

In our example above we’re not solving problems in an object oriented fashion. We’re simply using objects as containers of data. To really utilize the ideas of object oriented programming we need to move forward and learn about instance methods.

Exercise 52.2

Write a class called Player that exposes two public instance fields called X and Y of type int. The two fields should be initialized to the value 0. With the class in place you should be able to run the following code and get the same output.

Player p1 = new Player();
Player p2 = new Player();

p1.X += 2;
p1.Y = 3;

p2.X = -4;
p2.Y += 3;

Console.WriteLine($"({p1.X},{p1.Y})  ({p2.X},{p2.Y})");
